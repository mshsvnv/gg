## Обучение на Vimeo90K Triplet Dataset для интерполяции 2x

### 1. Генерация псевдо-меток оптического потока
Сначала запустите этот скрипт для генерации псевдо-меток:
```bash
python generate_flow.py
```

### 2. Запуск обучения
Начните обучение, выполнив одну следующюю команду
```bash
python train.py --model_name 'IFRNet' --epochs 100 --batch_size 6 --lr_start 1e-4 --lr_end 1e-5
```

## Бенчмарки для интерполяции 2x

Чтобы проверить время работы и количество параметров модели, запустите:
```bash
python benchmarks/speed_parameters.py --model_name 'IFRNet'
```

Чтобы проверить точность интерполяции на датасете Vimeo90K, запустите:
```bash
python benchmarks/eval_vimeo90k.py --model_name 'IFRNet'
```

## Экспериментальные модели и метрики

Все модели обучаются на Vimeo90K и валидируются на Vimeo90K.
Для каждого варианта замеряется производительность (скорость/параметры).

**Метрики обучения и валидации:**
- **Восстановление:** L2 (PSNR), SSIM
- **Структура/Границы:** Ternary (Census)
- **Консистентность потока:** Distill (flow)
- **Восприятие человеком:** LPIPS *(Выбрано как стандартная метика перцептивного качества для интерполяции)*

### 1. IFRNet RGB (Базовая)
Исходная архитектура модели, обрабатывающая полную информацию RGB.
- **Вход:** Полное разрешение каналов RGB.

### 2. IFRNet RGB/2 (Уменьшенная)
Облегченный вариант с вдвое меньшим количеством каналов, обрабатывающий меньше информации по сравнению с оригиналом.
- **Вход:** Полное разрешение каналов RGB.
- **Модификация:** Все каналы сети уменьшены в 2 раза.

### 3. IFRNet YUV420 (С субдискретизацией цветности)
Модель, использующая априорное знание о восприятии человеком (меньшая чувствительность к деталям цвета) для экономии вычислений. Яркость (Y) обрабатывается в полном разрешении, а цветность (UV) — в половинном.
- **Вход:** Формат YUV 4:2:0.
- **Модификация:** Y-канал обрабатывается в полном пространственном разрешении; UV-каналы обрабатываются в половинном разрешении.
